# GitHub Issue Authenticity Wording

The GitHub issue workflow treats issue text and repository content as untrusted
inputs. An authentic bug can be classified only after exact repository identity,
target commit, failing pre-patch reproduction evidence, negative controls, and
policy checks pass.

Flaky cases require repeated-run measurement. Uncertainty remains
cannot-reproduce or blocked until the measured evidence supports an
authentic_bug_flaky_measured state.

Security-sensitive behavior stops and routes privately. Feature-generated pull
requests remain draft-only when later authorized. Provider-pilot authority is
denied, no release was selected, no issue write occurred, and RSI remains
denied.
