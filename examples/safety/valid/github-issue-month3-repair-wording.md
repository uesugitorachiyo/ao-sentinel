# GitHub Issue Repair Wording

Month 3 repair work starts only after an authentic bug has failing pre-patch
reproduction evidence and a bounded repair workgraph. The repair keeps the
regression test enabled, runs post-patch verification, records exact rollback
digests, and accepts replay only when evidence digests match.

Feature-generated pull requests do not exist in Month 3 and remain draft-only
when later authorized. Provider execution is denied, no release or tag was
selected, and RSI remains denied.
